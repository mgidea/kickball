Let's Get Our Kick-on

What is the point of playing competitive kickball if you do not keep track of the winning team? I have created a Ruby program that collects the scores of each individual match and then reports who was the eventual winner.

The score for each game is recorded through a series of prompts. The data entry worker supplies the names of both teams and the final scores from the game. The program then invokes a method to report the winner and report who that winner is.

Many times there is more than one game that has taken place, so the program will ask whether the reporter wants to record any more games. The reporter is asked the same series of questions that requires the fill in the same fields of data. This process will continue until there are no games left to record. If the reporter responds "n" to the question "Would you like to provide another game? (Y/N)", the programs responds by printing out line by line the game number along with the corresponding winner. Here is a simulated example of output:

  • In game 1, the Bangers were the victors!
  • In game 2, the Bananas were the victors!
  • In game 3, the Haters were the victors!
  • In game 4, the Bangers were the victors!

    • I could not figure out how to reassign the variable name each time a new game was created. This should be easy enough to work through, but I wanted to submit a working program before I worried about secondary issues.

  2. Create validators on data input

    • Scores must be integers
    • Response to yes or no question must be y/n

  3. Produce more data in the final report
